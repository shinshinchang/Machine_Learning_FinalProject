## 1. 資料集整體概況

MovieLens 10M 原始資料集含 10,000,054 筆評分，經 **5-core 過濾**（每位使用者與每部電影均至少有 5 筆互動）後，保留 **9,998,816 筆**，流失率不到 0.02%，顯示原始資料已相當密集，5-core 在此資料集幾乎不造成截斷。

**表 1：資料集基本規模**

| 指標 | 數值 | 備註 |
|------|------|------|
| 總互動筆數 | 9,998,816 | 5-core 後 |
| 使用者數 | 69,878 | |
| 電影數 | 10,196 | 5-core 後 |
| 互動矩陣密度 | **1.403%** | 顯著高於電商場景 |
| 資料時間跨度 | 5,109 天 | 1995-01-09 至 2009-01-05（約 14 年） |
| 電影總數（原始） | 10,681 | 5-core 前 |
| k-core 門檻 | 5 | |

與 Amazon Beauty 資料集（密度 0.073%）相比，MovieLens 10M 的互動矩陣密度高達 **1.403%**，約為前者的 **19 倍**。這一差異反映了電影評分與美妝購物在行為模式上的根本差異：電影是消費性娛樂，評分門檻低、主動性強；而購買美妝品涉及金錢支出，互動更為稀少且目的性更強。

**對 CEINN 的意義**：較高的密度意味著 MovieLens 場景中冷門物品的曝光問題相對較輕——大多數電影都有足夠的互動記錄；但 §3.2 中的流行度偏差分析（下詳）將顯示，即便在較密集的資料集中，互動集中度仍然相當可觀，IPS 校正仍具必要性。

## 2. 序列行為特徵分析（§3.1 相關）

### 2.1 評分分布（Rating Embedding $E_r$）

> **圖 3.1：評分分布（$E_r$ 細粒度）**

MovieLens 10M 支援 **半星評分（0.5 至 5.0，共 10 個等級）**，此為其相較 Amazon 整數評分的重要差異。評分分布的統計特性如下：

| 指標 | 數值 |
|------|------|
| 半星評分使用比例 | 20.5%（`rating_half_star_share = 0.205`） |
| 五星評分比例 | 15.4%（`rating_share_5 = 0.154`） |
| 評分資訊熵 | **1.912 bits**（`rating_entropy = 1.912`） |

如圖所示，評分分布呈現**雙峰（bimodal）**結構：第一峰值在 3.0 附近（最高，約 240 萬筆），第二峰值在 4.0 附近（約 290 萬筆），五星評分（5.0）僅佔第三（約 155 萬筆）。與 Amazon Beauty 的極端右偏分布（57.7% 五星）截然不同，MovieLens 的評分分布更為均衡，資訊熵達 **1.912 bits**，遠高於 Beauty 的 1.205 bits。

值得關注的是，半星評分（0.5, 1.5, 2.5, 3.5, 4.5）的使用比例達 20.5%，代表約 200 萬筆評分使用了更精細的顆粒度，顯示用戶在表達情感強度時具有更高解析度的需求。

**對 CEINN 的意義**：高達 1.912 bits 的評分熵與雙峰分布，表明 §3.1.3 中 $E_r$ 評分嵌入在 MovieLens 場景下具有**顯著更高的信息量**。半星制的 10 級評分系統要求 $E_r$ 能夠區分 10 個等級的偏好強度（而非 Amazon 的 5 個整數等級），在嵌入設計上需留意：(1) 評分等級數量應設為 10；(2) 可考慮將半星評分值映射至連續嵌入空間，以捕捉其順序性（ordinal nature）。

### 2.2 使用者序列長度分布

> **圖 3.1：使用者序列長度分布（log scale）**

序列長度分布的統計數據如下：

| 百分位數 | 序列長度 |
|----------|----------|
| P50（中位數） | **69** |
| P90 | **335** |
| P99 | **1,057** |
| Max | **7,240** |

序列長度分布在對數尺度下呈現近似對稱的鐘形（bell-shape），主體集中在 50–200 的範圍，反映出相當一部分使用者擁有豐富的評分歷史。此分布與 Amazon Beauty（P50 = 6）形成鮮明對比：MovieLens 用戶的序列長度中位數高出 **11 倍以上**，反映電影消費的高頻次特性。

**對 CEINN 的意義**：序列長度 P50 = 69 意味著大多數使用者有充足的歷史記錄，能夠為 §3.1.4 因果遮罩自注意力機制（Causal-Masked Self-Attention）提供足夠的長距離依賴信號。然而，P90 = 335、P99 = 1,057 的長尾表明序列截斷閾值（truncation length）的選擇至關重要——建議設定為 200–500 之間（如 P90 = 335），以覆蓋主體使用者同時避免計算資源浪費。對於超長序列（如最大值 7,240），可採用滑動窗口（sliding window）策略進行處理。

### 2.3 時間間隔分布（Temporal Encoding $E_t$）

> **圖 3.1：相鄰評分時間間隔 Δt 分布（log scale）**

MovieLens 的時間間隔分布呈現與 Amazon Beauty 截然不同的極端特性：

| 指標 | 數值 |
|------|------|
| Δt < 1 分鐘的比例 | **78.9%**（`dt_share_under_1min = 0.789`） |
| Δt < 1 小時的比例 | **94.4%**（`dt_share_under_1hr = 0.944`） |
| Δt 的對數值域跨度 | **8.47 個數量級**（`dt_log10_range = 8.471`） |

如圖所示，Δt 分布呈現**雙峰（bimodal）**結構：第一個峰值集中在 1–100 秒（對應「連續批次評分」行為），第二個峰值出現在 $10^4$–$10^5$ 秒範圍（約 3–30 天，對應「週期性重返」行為），兩峰之間有明顯的低谷。

最引人注目的統計現象是：**78.9% 的相鄰評分發生在 1 分鐘之內**，94.4% 發生在 1 小時之內。這強烈暗示使用者的主要行為模式是「**批次回顧補評（retrospective batch rating）**」——用戶並非在觀看電影後立即評分，而是定期集中為多部已看電影打分，類似於「補課」行為。

**對 CEINN 的意義**：此發現對 §3.1.3 的時間編碼設計具有深遠影響：

1. **對數分桶的絕對必要性**：Δt 跨越 **8.47 個對數量級**（從不足 1 秒到超過 $10^8$ 秒，即約 3 年），是 Amazon Beauty 的 2.6 倍。任何線性尺度的時間表示都將完全失效，對數分桶是唯一可行方案。
2. **批次評分的時序語意污染問題**：由於 78.9% 的互動發生在 1 分鐘內，「相鄰評分的時間間隔」在此資料集中並非真實的「觀影間隔」，而是「補評的速度」。這意味著 $\Delta t_k$ 的語意與真實的偏好時間演進（temporal evolution of preference）存在嚴重的**觀測偏差**，是 CEINN 在 MovieLens 場景下時序建模的核心挑戰。
3. **建議**：可考慮在預處理階段對 Δt < 60 秒的連續評分事件進行「會話切分（session segmentation）」，以 session 為單位重新定義時間間隔，還原更接近真實消費時序的 Δt。

## 3. 流行度偏差與混淆變數分析（§3.2 相關）

### 3.1 流行度集中度——Lorenz 曲線

> **圖 3.2：流行度 Lorenz 曲線（Gini = 0.80）**

MovieLens 10M 的流行度 Gini 係數高達 **0.796**（`pop_gini = 0.796`），遠高於 Amazon Beauty（Gini = 0.50）。如 Lorenz 曲線所示：

- 最熱門的前 **20%** 電影吸引了 **84.4%** 的全部互動（`pop_head20_share = 0.844`）
- 其餘 80% 的長尾電影僅累積 15.6% 的互動

這意味著約 **8,000 部電影**（佔 10,196 的 78%）幾乎被大多數使用者忽視，曝光機會極度不均等。

**對 CEINN 的意義**：Gini = 0.80 的高度集中是 MovieLens 場景中曝光偏差問題最強烈的量化證據。相較於 Amazon Beauty（Gini = 0.50），此資料集的流行度偏差**更為嚴峻**，IPS 加權校正的潛在收益也相應更大。然而，這也帶來了更嚴峻的訓練挑戰——見下節分析。

### 3.2 逆傾向分數爆炸問題

在 MovieLens 場景中，由於缺乏外部排名資訊，§3.2.4 採用滾動互動頻率作為混淆變數代理 $Z_i(t)$。以此估計傾向分數後：

| 指標 | 數值 |
|------|------|
| $1/p_i$ 的 P99 | **6,972.8** |
| $1/p_i$ 的 P99.9 | **6,972.8** |
| $1/p_i$ 的最大值 | **6,972.8** |

逆傾向分數的最大值高達 **6,972.8**，且 P99、P99.9 與最大值完全相同，顯示存在**大量電影的傾向分數趨近於同一極小值**，形成顯著的地板效應（floor effect）。這一數值比 Amazon Beauty 的 86.2 高出**整整兩個數量級**。

**對 CEINN 的意義**：如此極端的逆傾向分數對 §3.2.7 的變異數縮減機制提出了最嚴苛的考驗。具體建議如下：

- **截斷閾值 $\tau$ 的設定**：建議遠低於最大值，如 $\tau \in [50, 200]$，視消融實驗結果調整。以最大值 6,972.8 為基準，即便截斷至 $\tau = 100$，仍可覆蓋大部分長尾電影的校正效果，而截斷比率（$\tau / \max = 1.4\%$）相當溫和。
- **自平滑 IPS（Self-Normalized IPS）的必要性**：在此場景下，未截斷的純 IPS 梯度的方差可達 Amazon Beauty 場景的 **$(6972.8/86.2)^2 \approx 6,556$ 倍**，訓練極不穩定，自平滑機制是必要的安全機制而非可選優化。

### 3.3 流行度的時間動態性

> **圖 3.2：前 100 熱門電影的流行度動態（20 個時間窗口）**

如圖所示，前 100 部電影在 20 個時間窗口（time windows）內的排名波動相當劇烈：許多電影的排名在不同窗口間的差距超過 100–200 位，部分甚至從接近第 1 位跌至第 400 位以外。

量化指標：前 100 部電影在各時間窗口中，排名偏離中位數超過 50 位的窗口比例平均為 **23.65%**（`rank_volatility_top100 = 0.2365`）。

**對 CEINN 的意義**：此發現為 §3.2.4 中 MovieLens 場景採用**動態混淆變數** $Z_i(t)$（滾動互動頻率）提供了直接的實證支持。若採用靜態流行度（如全局累積互動次數）作為 $Z$，將無法捕捉電影流行度的時間演變——例如老電影在特定時段的複興（如奧斯卡提名效應）或新片的冷卻效應。動態 $Z_i(t)$ 在此資料集中具有**不可替代的語意價值**，其所攜帶的信號量可由排名波動率 23.65% 加以佐證。

## 4. 行為摩擦力特徵分析（§3.3 相關）

### 4.1 電影類型分布（Genre Variable）

> **圖 3.3：類型分布（左）；每部電影的類型數量（右）**

MovieLens 10M 共包含 **19 個獨立類型**（`n_genres = 19`），分布如下：

- **頭部類型**：Drama（約 5,300 部）、Comedy（約 3,700 部）遠超其他類型。
- **中層類型**：Thriller、Romance、Action（各約 1,500–1,700 部）、Crime、Adventure、Horror（各約 1,000–1,150 部）。
- **尾部類型**：Film-Noir、Western（各約 275–285 部）、IMAX（約 30 部，極少）。

在類型數量方面，每部電影的類型數（cardinality）如下：

| 指標 | 數值 |
|------|------|
| P50（中位數） | **2 個類型**（`genres_per_movie_p50 = 2`） |
| 最大值 | **8 個類型**（`genres_per_movie_max = 8`） |

如圖所示，約有 4,000 部電影僅屬於 1 個類型，3,700 部屬於 2 個類型，隨後快速遞減；超過 5 個類型的電影極為少見。

**對 CEINN 的意義**：19 個類型的基數適中，支持 §3.3.4 中以類型二進位向量（Binary Vector）表示電影類型的設計。中位數 2 個類型意味著多數電影為**多類型混合**（而非單一類型），Jaccard 相似度在此可有效衡量類型重疊度（GenreRed），計算結果不會退化為 0 或 1 的極端值。然而，Drama 與 Comedy 的高度主導性（兩者合計覆蓋超過 40% 的電影）可能導致 GenreRed 計算中的類型偏差——建議在計算 Jaccard 時考慮 **IDF 加權**，降低高頻類型的影響力，以更精確地捕捉使用者真實的類型疲勞（genre fatigue）。

### 4.2 類型冗餘飽和效應前測（GenreRed Satiation Pre-test）

> **圖 3.3：GenreRed 飽和效應前測（驗證 §3.3.4）**

此前測以 Jaccard 相似度（下一部電影的類型向量 vs. 使用者近期觀影的類型集合）作為類型重疊度指標，分四個四分位桶分析，結果如下：

| Jaccard 桶 | 樣本數 | Jaccard 均值 | log10(Δt) 均值 | 平均評分 |
|-----------|--------|-------------|----------------|----------|
| (0.000, 0.143] | 302,561 | 0.098 | **1.575** | 3.481 |
| (0.143, 0.222] | 224,481 | 0.195 | 1.513 | 3.502 |
| (0.222, 0.333] | 284,764 | 0.288 | 1.476 | 3.498 |
| (0.333, 1.000] | 188,490 | 0.461 | **1.423** | 3.512 |

兩個關鍵觀察：

**（1）時間間隔（log10 Δt）隨 Jaccard 單調下降**：從最低重疊桶（1.575）到最高重疊桶（1.423），跨度 **0.152 個對數量級**（`satiation_log_dt_spread = 0.152`）。換言之，與使用者近期已看類型**越相似**的電影，下一次互動的間隔**越短**——用戶在「連續批次評分」的情境下確實傾向於連續評同類型電影。

**（2）評分方向不符合飽和假說**：理論上，若飽和效應存在，高類型重疊應導致評分下降（疲勞效應）。然而，觀測資料顯示評分從低重疊桶（3.481）到高重疊桶（3.512）呈**輕微上升**趨勢（`satiation_monotonic_rating = False`），與飽和假說方向相反。

**對 CEINN 的意義**：此前測結果呈現一個**複雜且矛盾**的信號，對 §3.3.4 的 GenreRed 設計具有重要的修正意涵：

1. **Δt 信號的混淆問題**：前文已指出 78.9% 的 Δt < 1 分鐘為批次補評行為。Jaccard 越高 → Δt 越短，這一關聯很可能不是「飽和導致拖延」，而是「批次補評時傾向連續評同類型」的行為模式所致。在此解讀下，Δt 作為飽和效應的代理指標在 MovieLens 中存在**嚴重的混淆**，無法直接採信。
2. **評分不支持飽和假說**：高 Jaccard 組的評分反而略高，提示「主動選擇同類型」的使用者可能是該類型的資深愛好者，評分普遍較高，而非因飽和而給出低分。這是一種典型的**選擇性偏差（selection bias）**。
3. **設計建議**：基於上述發現，§3.3.4 中 GenreRed 作為行為成本的設計需謹慎。建議將 GenreRed 的權重 $\beta_1$ 初始化為較小值，並在訓練過程中監控其學習動態；亦可考慮將 GenreRed 定義為「類型疲勞的懲罰項」，而非直接使用 Jaccard 值，需引入非線性轉換（如 sigmoid 壓縮）以避免對高度類型重疊的用戶過度懲罰。

### 4.3 標籤（Tags）資料的可用性分析

MovieLens 10M 附帶 95,580 筆標籤事件，相關統計如下：

| 指標 | 數值 |
|------|------|
| 標籤事件總數 | 95,580 |
| 被標記電影佔比 | **73.5%**（`tag_movie_coverage = 0.735`） |
| 有標記行為的使用者佔比 | **3.32%**（`tag_user_coverage = 0.033`） |
| 標籤詞彙量 | 15,241 |
| 單例標籤比例（僅出現 1 次） | **55.9%**（`tag_singleton_share = 0.559`） |

**對 CEINN 的意義**：標籤資料的利用價值極為有限，原因有三：

1. **使用者覆蓋率僅 3.32%**：超過 96% 的使用者從未使用過標籤功能，若引入標籤特徵，將造成嚴重的特徵稀疏問題。
2. **單例標籤佔 55.9%**：超過一半的標籤在整個資料集中僅出現一次，無法學習到有意義的嵌入表示。
3. **高詞彙量（15,241）+ 低品質**：維度龐大但信號雜訊比極低。

此分析**直接且充分地支持** §3.3.4 的設計選擇——「聚焦於高質量的電影類型（Genres）與時間序列，摒棄社群標籤（Tags）」。標籤在 MovieLens 10M 中的可用性遠低於其名義覆蓋率所暗示的水準，排除標籤是明智且有據可查的工程決策。

## 5. 時間分布與資料分割可行性

> **圖 X：每年互動量（資料分割可行性評估）**

MovieLens 10M 的時間分布（1995–2009）呈現獨特的**非單調波動**模式，與 Amazon Beauty 的指數型成長截然不同：

| 年份 | 互動筆數 | 備註 |
|------|----------|------|
| 1995 | 3 | 幾乎無資料 |
| 1996 | 1,047,618 | 第一波高峰 |
| 1997 | 459,941 | 下降 |
| 1998 | 202,091 | 谷底 |
| 1999 | 788,788 | 回升 |
| 2000 | **1,271,610** | **全期最高** |
| 2001 | 759,132 | 下降 |
| 2002–2004 | ~688,650 均值 | 相對穩定 |
| 2005 | **1,177,203** | 第三波高峰 |
| 2006–2008 | ~745,929 均值 | 穩定期 |
| 2009 | 14,549 | 截斷（資料至 1 月） |

資料量較為均衡，各年份（2002–2008）均維持在 580,000–1,270,000 筆之間，無明顯單向趨勢。2009 年僅涵蓋前 5 天（截止 1 月 5 日），資料量極少。

**對 CEINN 的意義**：

1. **時序切分優於隨機切分**：儘管年際分布相對均衡，推薦場景的本質仍要求嚴格的時間因果性。建議採用 leave-one-out 時序切分（以使用者最後一筆互動為測試集，倒數第二筆為驗證集），確保不發生未來資訊洩漏（Future leakage）。
2. **2009 年資料應排除或單獨處理**：僅 14,549 筆互動且截止於 1 月 5 日，若不排除，將使 2009 年的評分看似該年「冷清」，實際上只是截斷效應。
3. **動態混淆變數 $Z_i(t)$ 的訓練支持**：各年均有充足的互動量，支持按時間窗口動態計算滾動互動頻率作為 $Z_i(t)$，不存在早期資料過稀而無法估計的問題（相較於 Amazon Beauty 早期僅數百筆的情形）。
4. **無嚴重分布偏移（Distribution Shift）的時段**：各年互動量的標準差相對均衡，訓練集（1995–2007 年主體）與驗證/測試集之間的時序分布差異不太可能造成嚴重的協變數偏移（covariate shift），有利於評估指標的可靠性。

### 6.3 MovieLens 場景的核心挑戰與風險

本 EDA 揭示了 CEINN 在 MovieLens 10M 場景下需特別謹慎處理的三項核心風險：

**風險一：批次補評行為對時序建模的污染**
78.9% 的 Δt < 1 分鐘顯示大多數「序列」並非真實的時序偏好演進，而是補評行為的人工產物。這可能導致 $E_t$ 的時間編碼學習到的是「補評速度」而非「偏好演化節奏」，對潛在狀態 $h_t$ 的因果解釋性構成威脅。建議在預處理階段引入 session segmentation（如以 30 分鐘無活動作為 session 邊界）。

**風險二：逆傾向分數的極端爆炸**
最大逆傾向分數 6,972.8 在訓練中若未適當截斷，將導致梯度方差暴增，模型無法穩定收斂。此風險在 MovieLens 場景中比 Amazon Beauty **嚴重約 80 倍**，是兩資料集在工程實現上最大的差異點。

**風險三：GenreRed 飽和假說未獲直接確認**
前測結果顯示類型重疊度（Jaccard）與評分之間的關係與飽和假說預期方向相反，飽和效應在 MovieLens 資料中的存在性尚未獲得直接的觀測證據。建議設計更嚴格的反事實實驗（如控制使用者觀影頻率），以確認 GenreRed 項的語意有效性。
